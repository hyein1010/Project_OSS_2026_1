## 1. Introduction
본 문서는 Conceptualization 단계의 문서에 이어지는 Analysis 단계의 문서다.
### 1.1 Summary
최근 현대인들의 일상에서 가장 빈번하게 발생하는 고민 중 하나는 수많은 선택지 속에서 식사 메뉴를 결정하는 것이다. 
<br>이에 따라 결정 장애를 겪는 사용자들은 메뉴를 고르는 데 상당한 시간과 에너지를 소모하고 있으며 이는 곧 의사결정 피로로 이어진다.
<br>때문에 본 문서에서는 사용자의 고민을 해결해 줄 수 있는 ‘메뉴 랜덤 뽑기 및 맛집 지도’ 서비스를 제안한다.
<br>시스템이 사용자의 위치, 날씨, 개인적 취향 등을 고려하여 최적의 메뉴를 추천하고 주변 맛집 정보를 시각적으로 제공함으로써
<br>사용자는 훨씬 더 빠르고 즐겁게 식사 결정을 내릴 수 있을 것으로 기대한다.

### 1.2 Business Goals
- 사용자가 결정 고민 없이 신속하게 식사 메뉴를 선택할 수 있도록 랜덤 추천 기능을 제공해야 한다.
- 추천된 메뉴를 판매하는 주변 음식점의 위치를 지도상에 직관적으로 표시해야 한다.
- 사용자의 과거 선호도나 즐겨찾기 데이터를 기반으로 개인화된 추천 리스트를 관리해야 한다.
- 영업시간, 거리, 가격대 등 구체적인 필터링 조건을 통해 실용적인 검색 결과를 제공해야 한다.

### 1.3 Technical Goals
- 사용자의 실시간 GPS 좌표를 활용하여 정확한 위치 기반 서비스(LBS)를 제공해야 한다.
- 외부 API(OpenWeatherMap 등)와 연동하여 날씨 상황에 맞는 메뉴 추천 알고리즘을 구현해야 한다.
- 지도 API를 통해 음식점의 위치 및 상세 정보를 실시간으로 렌더링할 수 있어야 한다.
- 영업시간 데이터를 서버 시간과 대조하여 현재 방문 가능 여부를 정확히 판별해야 한다.
- 모바일 환경에 최적화된 사용자 친화적인 UX/UI를 구축하여 접근성을 높여야 한다.

## 3. Domain analysis 
<img width="892" height="612" alt="image" src="https://github.com/user-attachments/assets/090eca6d-d336-4fe8-9010-9001994fefc7" />

---
### 1) View
사용자 인터페이스를 담당하며 메뉴 추천 결과, 맛집 지도, 리뷰 작성 폼 등을 화면에 출력한다. 
사용자의 클릭 이벤트를 받아 ServiceController로 전달하고 처리 결과를 화면에 반영하는 독립적인 GUI 모듈이다.

### 2) ServiceController
View에서 전달받은 요청(메뉴 추천, 식당 검색 등)을 해석하고 DataController를 통해 필요한 정보를 가공하여 View에 전달한다. 
비즈니스 로직의 흐름을 제어하는 핵심 모듈이다.

### 3) DataController
사용자 정보, 식당 데이터, 즐겨찾기 목록 등 시스템의 모든 데이터베이스 접근을 관리한다. 
데이터의 읽기, 쓰기, 수정, 삭제(CRUD) 기능을 수행하며 필요 시 외부 API 클래스에 데이터를 요청한다.

### 4) MapInterface
DataController의 요청을 받아 지도 API(카카오/구글 등)를 연동한다. 
현재 좌표를 기반으로 지도 데이터를 렌더링하고 식당 위치에 마커를 생성하여 결과를 DataController에 응답한다.

### 5) RecommendationEngine
ServiceController의 요청을 받아 메뉴 추천 알고리즘을 수행한다. 
현재 날씨, 시간, 사용자의 필터 설정값을 분석하여 최적의 메뉴를 선별한 뒤 결과를 응답한다.

### 6) InfoValidator
DataController가 관리하는 식당 정보 중 영업시간을 현재 서버 시간과 대조한다. 
방문 가능 여부를 판별하여 영업 종료된 가게에 대한 뮤트(Mute) 상태값을 응답한다.

### 7) ReviewManager
사용자가 입력한 리뷰 데이터(사진, 텍스트)의 유효성을 검사하고 저장한다. 
부적절한 단어 필터링 기능을 수행하며 평점 평균을 계산하여 DataController에 전달한다.

### 8) AuthManager
회원가입 및 로그인 시 본인인증 API를 호출하고 사용자의 계정 권한을 확인한다. 
인증 성공 여부와 세션 정보를 관리하여 시스템 보안을 유지한다.
